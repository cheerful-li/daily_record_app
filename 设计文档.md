# 个人管理网站设计文档

## 1. 项目概述

### 1.1 背景与目标

个人管理网站旨在提供一站式的个人成长与任务管理平台，通过微习惯培养、事件记录、任务管理等功能，帮助用户建立良好的生活习惯，提高工作效率，促进个人成长。

### 1.2 核心价值

- 习惯养成：通过微习惯的持续追踪，帮助用户形成良好的日常行为模式
- 时间管理：通过待办事项的分类管理，提高时间利用效率
- 回忆积累：记录生活点滴与个人成长轨迹，积累复盘素材
- 关系维护：帮助用户维护个人社交网络
- 创意积累：收集并整理灵感与想法

## 2. 系统架构

### 2.1 技术栈

- **运行时环境**：Bun
- **构建工具**：Vite
- **前端框架**：React
- **语言**：TypeScript
- **UI组件**：shadcn/ui
- **状态管理**：MobX
- **数据存储**：IndexedDB

### 2.2 系统结构

```
客户端应用 (SPA)
    │
    ├── 用户界面 (React + shadcn/ui)
    │
    ├── 状态管理 (MobX)
    │
    └── 本地存储 (IndexedDB)
```

### 2.3 数据存储设计

采用IndexedDB实现客户端本地存储，主要数据表设计：

1. **微习惯表(Habits)**：
   - id: 唯一标识符
   - name: 习惯名称
   - description: 习惯描述
   - frequency: 频率(daily/weekly/monthly)
   - active: 是否激活
   - createdAt: 创建时间
   - updatedAt: 最后更新时间

2. **打卡记录表(CheckIns)**：
   - id: 唯一标识符
   - habitId: 关联的习惯ID
   - date: 打卡日期
   - status: 状态(completed/half-completed/skipped)
   - note: 备注
   - createdAt: 创建时间
   - updatedAt: 最后更新时间

3. **生活点滴表(LifeMoments)**：
   - id: 唯一标识符
   - title: 事件标题
   - description: 事件描述
   - date: 事件日期
   - tags: 标签数组
   - attachments: 附件(可选)
   - createdAt: 创建时间
   - updatedAt: 最后更新时间

4. **待办事项表(Tasks)**：
   - id: 唯一标识符
   - title: 任务标题
   - description: 任务描述
   - type: 类型(work/growth)
   - status: 状态(pending/in-progress/completed)
   - dueDate: 截止日期(可选)
   - priority: 优先级
   - createdAt: 创建时间
   - updatedAt: 最后更新时间

5. **灵感想法表(Ideas)**：
   - id: 唯一标识符
   - content: 内容
   - date: 记录日期
   - category: 分类
   - createdAt: 创建时间
   - updatedAt: 最后更新时间

6. **社交关系表(Relationships)**：
   - id: 唯一标识符
   - name: 姓名
   - category: 分类(家人/朋友/同事等)
   - lastContact: 上次联系时间
   - nextContact: 下次联系时间(提醒)
   - notes: 备注
   - createdAt: 创建时间
   - updatedAt: 最后更新时间

## 4. 用户界面设计

### 4.1 总体布局

采用响应式设计，主要布局结构:

- 顶部导航栏：主要模块切换
- 侧边栏：当前模块的子功能
- 主内容区：功能主界面
- 底部状态栏：系统状态、同步状态等信息

### 4.2 设计风格

- 使用shadcn/ui组件库，保持一致的设计语言
- 主题颜色：主色调+辅助色系
- 支持亮色/暗色模式切换
- 简洁明了的UI设计，减少视觉干扰
- 重点突出关键信息和操作

### 4.3 交互设计

- 拖放操作：任务排序、优先级调整
- 手势操作：适配移动设备
- 快捷键支持：提高操作效率
- 实时反馈：操作后立即显示结果

## 5. 数据流设计

使用MobX进行状态管理，主要Store设计：

1. **HabitStore**: 管理微习惯数据
2. **LifeMomentStore**: 管理生活点滴数据
3. **TaskStore**: 管理任务数据(工作/成长)
4. **RelationshipStore**: 管理社交关系数据
5. **IdeaStore**: 管理灵感想法数据
6. **UIStore**: 管理UI状态(主题、布局等)

数据流向:
1. UI事件触发 -> Action调用
2. Action执行 -> 更新Store状态
3. Store状态变化 -> 触发UI重新渲染
4. Store状态变化 -> 同步至IndexedDB

### 阶段四：优化与扩展

## 7. 技术实现要点

### 7.1 本地存储优化
- 使用IndexedDB事务确保数据一致性
- 实现简单的数据迁移机制，支持未来版本升级
- 定期数据清理与优化
- 自动维护时间戳(createdAt/updatedAt)，记录数据变更历史

### 7.2 状态管理
- 使用MobX进行细粒度的响应式状态管理
- 实现Store间的数据依赖关系
- 优化组件重渲染效率

### 7.3 UI组件复用
- 构建可复用的基础组件
- 实现主题定制与切换
- 确保组件的可访问性(a11y)

### 7.4 离线使用支持
- 完全基于客户端实现，无需后端服务
- 确保所有功能在离线环境下可用

## 8. 非功能性需求

### 8.1 性能目标
- 首次加载时间 < 2秒(优化后)
- 交互响应时间 < 100ms
- 数据操作响应时间 < 200ms

### 8.2 兼容性
- 支持主流现代浏览器(Chrome, Firefox, Safari, Edge)
- 响应式设计，适配桌面与移动设备

### 8.3 可扩展性
- 模块化设计，便于未来功能扩展
- 预留数据同步机制接口，为未来多设备同步预留可能

## 9. 总结

本设计文档概述了个人管理网站的核心功能与技术实现方案。通过微习惯打卡、事件记录、任务管理等模块的紧密集成，为用户提供全方位的个人成长与管理工具。采用现代前端技术栈，确保良好的用户体验与系统性能。

项目采用渐进式开发策略，先实现核心功能，后续逐步添加高级功能，确保项目可持续迭代与优化。
